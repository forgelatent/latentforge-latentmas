# INCIDENT_2026-04-18 — Benchmark Data Contamination

**Status:** RESOLVED (remediation complete April 19, 2026)
**Severity:** High (affected strategic decisions, not external-facing)
**Impact:** Internal only — no external claims were published
**Classification:** Data integrity / trust architecture failure

---

## 1. Summary

Between March 29 and April 18, 2026, the LatentForge benchmark pipeline generated a narrative of multi-day prediction-market outperformance that did not exist. The "crowd baselines" the system compared itself against were hand-written hypothetical numbers in a seed file, not live Polymarket data. The swarm wasn't beating live markets — it was beating numbers the founder had typed into a JSON file on March 29.

The narrative propagated across the benchmark report (v0.1), daily commercialization agent outputs, the Rain grant draft, the Polymarket outreach draft, BRAIN.md, and research suggestions before being detected on April 18 during live-data verification.

## 2. Root Cause

**Primary cause:** experiments/benchmark/policy_markets_seed.json was manually authored on March 29, 2026 (git commit 6712f67: "Day 1 benchmark: policy markets selector + text baseline running") with hypothetical policy questions and made-up current_price values. The text swarm loaded this file as if it were live Polymarket data, via markets = json.load(open(SEED_FILE)) in 03_text_swarm.py. No live API call was ever wired in.

**Systemic cause — Social Proof Loop:** The three decision engines (Founder / Systems / Divergent) repeatedly synthesized each other's interpretations of the swarm output without returning to the raw data. Each engine's confidence compounded the next engine's confidence. Uncertainty was converted to conviction without new evidence being introduced. This is the specific failure mode the three-engine system was supposed to prevent.

**Contributing factors:**
- No evidence-tier tagging on claims entering BRAIN.md
- No source-type tagging on benchmark outputs
- No pre-generation data-freshness gate on the benchmark updater
- Static current_price values never changed across 20 days, but this signal was not flagged
- com.latentforge.polymarket-pull.plist was never created, so no live Polymarket data was ever pulled automatically (detected separately April 19 by Grok)

## 3. Invalidated Claims

The following claims are officially retracted. They must not appear in any external communication, grant application, outreach, investor material, or strategic document:

- "20-day AI regulation divergence (swarm 20-22% vs crowd 31%)" — the 31% crowd baseline was fictional
- "45% Brier improvement over naive baseline" — source data was fictional
- "Sustained directional divergence from crowd consensus" — no live crowd existed to diverge from
- "Outperformed live markets" — no live market comparison was being made
- "Beat the crowd on policy markets" — no crowd baseline was real
- "15-day swarm consistency against crowd" — closed-loop echo chamber against fictional data; salvageable only as a system-reliability log, never as a forecasting signal
- Any Brier Skill Score vs crowd, calibration subset metrics, or divergence scores calculated before April 19, 2026

## 6. Forward Protocol

Effective immediately, the following rules govern all claims entering BRAIN.md, outreach, or strategic documents:

**Evidence tiers (required tag on every claim):**
- Tier 0: Intuition / hypothesis
- Tier 1: Observed once, not audited
- Tier 2: Reproducible internally, source verified
- Tier 3: Externally defensible, raw-to-report audited

Strategic decisions and external communications require Tier 2 evidence minimum. Tier 3 is required for citations in grant applications, investor materials, or outreach.

**Source-type tag (required on every metric):**
- [LOCAL_LOG] — verified by script output on local machine
- [API_VERIFIED] — direct pull from Polymarket / Kalshi / other external API
- [DERIVED] — calculated from above; computation must be shown

**Three-engine protocol (anti-Social-Proof-Loop):**
- No engine may synthesize another engine's conclusions into BRAIN.md without the Founder Engine explicitly promoting it
- When a claim is disputed or looks anomalous, the protocol is: git history check → raw data inspection → timestamp sanity check → source freshness check → independent recomputation, before any new synthesis is allowed
- Silent Operator mode: each engine provides independent raw feedback; synthesis happens only at the Founder level

**When this document should be cited:** Any time pre-April 19 benchmark data, "vs crowd" claims, Brier score improvements, or divergence numbers appear in conversation, outreach, or internal reasoning, reference this incident record and treat those claims as retracted.
