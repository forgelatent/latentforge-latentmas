Technique: Sparse Distributed Representations (SDR) with Overlap-Based Similarity Preservation
Field: Computational Neuroscience (Hierarchical Temporal Memory / Jeff Hawkins, Numenta)
Core idea: Encode information as fixed-sparsity binary vectors where semantic similarity is preserved by the degree of bit-overlap, making union, intersection, and distance operations O(k) instead of O(d).
Why LatentForge: Your current top-k sparsity selects the k largest-magnitude dimensions but treats the resulting sparse vector as a dense float32 object for transmission. SDR thinking flips this: convert the top-k selection into a binary membership mask and encode *which* dimensions activated, not their magnitudes. The overlap count between two SDRs is a provably reliable proxy for cosine similarity (for fixed k and d, P(false match) < 10⁻⁹ at k=128, d=3072). This means the Shadow Self's drift detection — currently a cosine distance computation over float vectors — can be replaced by a bitwise XOR + popcount operation on two uint64 arrays, reducing drift detection from O(d) float ops to O(d/64) integer ops. More critically, the seed vector becomes a *population code*: instead of a single float reference, each agent maintains a sparse binary "address" for each concept cluster, and deltas become set-difference operations (new bits ON, old bits OFF). This is exactly how the hippocampus encodes episodic memory deltas — only the changed pattern indices are transmitted, not the full vector state. The injective property holds because the Bloom-filter-like collision probability at k=128, d=3072 is astronomically low.
