Technique: Predictive Coding Residuals (Hierarchical Error Propagation)
Field: Computational Neuroscience / Predictive Processing Theory
Core idea: Transmit only the prediction error between what a receiving agent *expects* the latent state to be and what it actually is, compressing by encoding surprise rather than raw deltas.
Why LatentForge: The brain doesn't transmit raw sensory data — it transmits residuals against a generative model's predictions. Applied to LatentForge: instead of sending a top-k sparse delta from the shared seed, each agent maintains a lightweight *predictive model* of where the other agent's latent state is heading (e.g., an exponential moving average or low-order autoregressive model over recent delta sequences). The sender then encodes only the residual between the predicted next state and the actual next state. Because agent latent trajectories have strong temporal autocorrelation during coherent reasoning chains, this residual will be dramatically sparser than the raw delta — potentially reducing required k from 128 to 32-64 with equivalent divergence preservation. The shared seed vector already provides the zeroth-order prediction; this extends it to first- and second-order. Critically, the Shadow Self drift monitor can be repurposed as the prediction model, giving it dual governance + compression roles.

SUGGESTION 2
Technique: Sparse Distributed Representations via Population Coding Geometry (SDR)
Field: Neuroscience / HTM Theory (Numenta) + Compressed Sensing
Core idea: Encode latent deltas as fixed-sparsity binary activation patterns where the *identity* of active dimensions carries meaning, not their magnitudes — making the representation both compressible and collision-resistant by construction.
Why LatentForge: Hierarchical Temporal Memory research established that cortical columns use sparse distributed representations where roughly 2% of neurons are active at any time, and the combinatorial capacity of *which* neurons fire vastly exceeds the capacity of analog magnitude encoding. LatentForge's top-k sparsity already selects indices, but treats their magnitudes as the primary signal. Inverting this: quantize magnitudes to 1-bit (sign only) or 2-bit (sign + magnitude tier), and instead spend the bit budget on *which* dimensions are active with higher index precision. A k=128 selection over d=4096 dimensions with 2-bit magnitudes costs 128×(12+2) = 1,792 bits. An SDR approach with k=64 active dimensions, full 12-bit index encoding, and 1-bit sign costs 64×13 = 832 bits — 2x further compression — while preserving the geometric separability of divergent states because SDR overlap directly measures semantic similarity. The Shadow Self governance layer can use Hamming distance on SDR patterns as a cheap drift alarm, replacing cosine similarity computation.

SUGGESTION 3
Technique: Genomic Differential Compression via Positional Run-Length Encoding on Sorted Index Maps
Field: Genomics / Bioinformatics (variant calling, VCF format)
Core idea: Represent latent deltas as a sorted list of (position, change) pairs — analogous to a genomic variant call file — then compress the position list using run-length encoding on the *gaps* between active indices, which are geometrically distributed and highly compressible.
Why LatentForge: The VCF (Variant Call Format) in genomics solves an identical mathematical problem: a reference genome exists (the shared seed vector), and each individual organism is represented as a sparse set of *mutations* from that reference. VCF stores only changed positions with delta values. Crucially, genomics discovered that inter-variant gap distances follow a near-geometric distribution, which means Elias gamma or Fibonacci coding of the gaps achieves near-optimal compression without modeling. Applied to LatentForge: after top-k selection, sort the active indices. Encode the first index in full (e.g., 12 bits for d=4096), then encode subsequent indices as gaps using Elias-gamma coding. Empirically in genomics, this reduces index storage by 40-60% over naive fixed-width encoding. Combined with the SDR suggestion above (sign-only magnitudes), this could push total delta payload below 512 bits for k=64 — roughly 50x compression vs. float32 dense vectors. The technique is entirely lossless on the selected indices, preserving the injective property LatentForge requires.
